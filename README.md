## О проекте

Это моя курсовая работа по классическому машинному обучению. Задача взята из реального научного контекста: есть данные о 1001 химическом соединении, которые тестировали на активность против вируса гриппа. Нужно было построить модели машинного обучения, которые помогут предсказывать эффективность и безопасность новых соединений — это позволяет ускорить поиск потенциальных лекарств, не синтезируя каждое соединение вручную.

Три целевые переменные:

| Переменная | Смысл | Хорошее значение |
|------------|-------|-----------------|
| **IC50 (mM)** | Минимальная ингибирующая концентрация против вируса | Чем **ниже**, тем активнее |
| **CC50 (mM)** | Концентрация, при которой погибает 50% здоровых клеток | Чем **выше**, тем безопаснее |
| **SI** | Индекс селективности: `SI = CC50 / IC50` | Чем **выше**, тем лучше; SI > 8 — перспективное соединение |

Признаки — 210 молекулярных дескрипторов (числовые характеристики молекул, вычисленные через RDKit).

## Методология

### Предобработка (единая для всех задач)
- Пропуски: `SimpleImputer(strategy='median')` — 36 NaN в 3 строках из 12 признаков
- Целевые переменные в регрессии: `log1p(target)` при обучении, `expm1()` для метрик
- Масштабирование: `StandardScaler` для линейных моделей и SVC/KNN
- Разбивка: train/test = 80/20, `stratify=y` в задачах классификации
- Кросс-валидация: `StratifiedKFold(5, shuffle=True)` для классификации

### Модели (сравнивались в каждой задаче)
- Линейные: `LinearRegression`, `Ridge`, `Lasso`, `ElasticNet`, `LogisticRegression`
- Метрические: `KNeighborsClassifier`, `SVR`, `SVC`
- Ансамблевые: `RandomForestRegressor/Classifier`, `ExtraTreesRegressor/Classifier`
- Бустинг: `GradientBoostingRegressor/Classifier`, `HistGradientBoosting*`, `CatBoost` (доп. проверка в IC50/SI по рекомендации лектора)

Для каждой задачи: базовые модели → `GridSearchCV` для топ-5 → финальное сравнение.

В EDA дополнительно проверяется статистическая значимость корреляций (низкокоррелированные признаки намеренно не удаляются) и убираются 18 константных признаков через `VarianceThreshold`. В ноутбуке IC50 добавлена оценка области применимости модели (Applicability Domain) — надёжность предсказания по расстоянию до обучающих данных.

## Результаты

### Регрессия

| Задача | Лучшая модель | R² (тест) | CV R² |
|--------|--------------|-----------|-------|
| IC50 | Random Forest (tuned) | **0.473** | 0.415 |
| CC50 | Extra Trees (tuned)   | **0.491** | 0.428 |
| SI   | Random Forest (tuned) | **0.334** | 0.308 |

### Классификация

| Задача | Лучшая модель | ROC-AUC | CV-AUC |
|--------|--------------|---------|--------|
| IC50 > медианы | Random Forest (tuned) | **0.784** | 0.822 |
| CC50 > медианы | Extra Trees (tuned)   | **0.864** | 0.835 |
| SI > медианы   | SVC (rbf)             | **0.727** | 0.718 |
| SI > 8         | Extra Trees bal. (tuned) | **0.751** (AP=0.693) | 0.740 |

> Задача SI > 8 — несбалансированная (64%/36%). Использовал `class_weight='balanced'` и оценивал через Average Precision (AP=0.693 у ET balanced tuned), а не только ROC-AUC. Лучший по ROC-AUC на тесте — Gradient Boosting (default, 0.757), но по CV-AUC и AP предпочтительнее Extra Trees balanced (tuned) — именно его рекомендую для скрининга.

- **ExtraTrees и RandomForest** — лучшие методы на 6 из 7 задач. Нелинейность молекулярных дескрипторов важна.
